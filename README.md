# ⛪ 새롬교회 임직감사예배 모바일 초청장 랜딩페이지

본 프로젝트는 **2026년 7월 5일(주일) 오후 2시**에 드려지는 **새롬교회 임직감사예배**의 모바일 및 웹 초청장 랜딩페이지입니다.
웹 디자인 트렌드와 현대적 감성을 담아 프리미엄 스타일로 제작되었으며, 모바일 환경에 완벽히 대응(반응형 디자인)합니다.

## ✨ 주요 기능 및 특징

1. **프리미엄 비주얼 디자인**
   - 새롬교회 2026년 표어인 **"물가로, 말씀으로, 생명으로"** 로고와 심볼의 딥 네이비(`#0E3A82`), 블루, 코랄 오렌지 그라데이션 컬러 시스템을 적용했습니다.
   - 글래스모피즘(Glassmorphic) 플로팅 네비게이션 바와 부드러운 스크롤 애니메이션(Scroll Reveal)을 적용해 세련된 사용자 경험을 선사합니다.
   
2. **최적화된 미디어 리소스**
   - 원본 14MB 크기의 초고화질 임직자 프로필 사진 및 약도 이미지를 가로 폭 조절과 리샘플링 가공 작업을 거쳐 **화질 저하 없이 총 1.1MB 크기로 최적화 압축**하여, 모바일 데이터 통신에서도 끊김 없이 초고속 로딩이 가능합니다.

3. **실시간 디데이 카운트다운**
   - 예배 일시(2026년 7월 5일 오후 2시)까지 남은 시간을 일, 시간, 분, 초 단위로 실시간 계산하여 표시합니다.

4. **스마트 캘린더 등록 연동**
   - **구글 캘린더**에 즉시 일정을 자동 추가할 수 있는 연동 버튼을 지원합니다.
   - 아이폰, 아웃룩 등에서 일정 추가가 가능한 **표준 캘린더 파일(.ics)**의 실시간 브라우저 내 다운로드 생성을 지원합니다.

5. **지도 및 오시는 길 내비게이션**
   - 가독성 높은 맞춤형 약도 이미지를 제공합니다.
   - **카카오맵** 및 **네이버 지도**로 바로 연결하여 하차 정류장과 길찾기 내비게이션을 즉시 실행할 수 있는 바로가기 버튼을 제공합니다.
   - 원클릭 **주소 복사(Copy Address) 기능**과 주소 복사 완료 시 하단에 노출되는 애니메이션 토스트 알림을 탑재했습니다.

6. **축하 방명록 (Guestbook)**
   - 임직자분들에게 따뜻한 축하와 기도를 전할 수 있는 방명록 기능입니다.
   - **축하 스티커(🙏 축하합니다, ✨ 축복합니다, 🌱 응원합니다, 💖 평생동행)**를 선택하여 메시지를 작성할 수 있습니다.
   - 브라우저의 `localStorage`를 활용하여 메시지가 즉각적이고 영구적으로 유지 및 렌더링되도록 구현했습니다. 작성 시 설정한 비밀번호를 통해 본인이 작성한 메시지를 개별 삭제할 수도 있습니다.
   
7. **기쁜 소식 공유 기능**
   - 카카오톡 공유 기능 및 링크 복사 기능을 갖추고 있습니다.

## 💻 로컬에서 테스트하기

프로젝트 폴더 내 `index.html`을 더블 클릭하여 웹 브라우저에서 바로 열어볼 수 있습니다. 

로컬 웹 서버 형태로 더욱 완벽하게 테스트하려면 터미널에서 다음 명령어를 실행해 보세요:

```bash
# Python을 사용하는 경우
python3 -m http.server 8000

# Node.js (npx)를 사용하는 경우
npx http-server -p 8000
```
그 후 브라우저에서 `http://localhost:8000`에 접속하면 테스트할 수 있습니다.

## 🚀 깃허브(GitHub Pages)를 통해 누구나 접속할 수 있게 배포하기

깃허브 페이지를 이용하면 완전 무료로 나만의 단독 도메인과 웹 주소를 생성해 모바일 초청장을 배포할 수 있습니다.

### [1단계] 깃허브 저장소(Repository) 생성
1. [GitHub](https://github.com)에 로그인합니다.
2. 우측 상단의 `+` 버튼을 누르고 **New repository**를 클릭합니다.
3. Repository name에 `saerom-invitation` 과 같이 입력합니다.
4. **Public**으로 설정(반드시 필요)한 뒤, 다른 옵션은 건드리지 않고 **Create repository**를 클릭합니다.

### [2단계] 코드 올리기 (Git 명령어 입력)
본 프로젝트 폴더 내에서 터미널을 열고 다음 명령어를 순서대로 실행합니다:

```bash
# 1. git 초기화
git init

# 2. 파일 추가 및 커밋
git add .
git commit -m "feat: 새롬교회 임직감사예배 초청장 릴리즈"

# 3. 브랜치명을 main으로 설정
git branch -M main

# 4. 내 깃허브 저장소 주소 연결 (username 부분에 본인의 깃허브 아이디 입력)
git remote add origin https://github.com/본인아이디/saerom-invitation.git

# 5. 코드 올리기
git push -u origin main
```

### [3단계] GitHub Pages 웹 호스팅 활성화
1. 방금 코드를 올린 깃허브 저장소 화면에서 상단의 **Settings** 메뉴로 이동합니다.
2. 좌측 사이드바의 **Code and automation** 카테고리 아래에 있는 **Pages** 탭을 클릭합니다.
3. Build and deployment > Source 항목이 **Deploy from a branch**로 되어 있는지 확인합니다.
4. Branch 항목에서 `None`을 클릭하여 `main`으로 변경하고, 우측의 `/(root)`는 그대로 둔 채 **Save** 버튼을 누릅니다.
5. 약 1~2분 뒤 페이지를 새로고침하면 상단에 다음과 같은 실제 웹 주소가 생성됩니다:
   `👉 https://본인의깃허브아이디.github.io/saerom-invitation/`

이 주소를 복사하여 성도분들이나 지인분들께 카카오톡, 문자 메시지 등으로 전달하시면 누구나 바로 모바일에서 접속할 수 있습니다!

## 🛠️ 추가 개발자 커스터마이징 가이드

### 카카오톡 공유 기능 활성화
현재 공유 버튼은 링크 복사로 대체 동작하고 있으나, 카카오톡 메시지 카드 형태로 연동하려면 다음과 같이 설정합니다:
1. [Kakao Developers](https://developers.kakao.com/)에서 애플리케이션을 생성하고 `JavaScript 키`를 발급받습니다.
2. `index.html` 의 `<head>` 내에 카카오 SDK 스크립트를 추가합니다:
   ```html
   <script src="https://t1.kakaocdn.net/kakao_js_sdk/2.4.0/kakao.min.js" integrity="sha384-mfn7nZCxn7y0RLfg5d95g1NPDJ1z4C2v1RyeG11R2A+61h23PZ6y0zL0P1a" crossorigin="anonymous"></script>
   ```
3. `js/main.js` 파일 최상단에 다음 코드를 추가합니다:
   ```javascript
   Kakao.init('발급받은_자바스크립트_키');
   ```

### 영구적인 방명록 데이터베이스(DB) 연결
현재 방명록은 작성한 기기의 브라우저(`localStorage`)에 저장되어 데모용으로는 적합하나, 다른 사람이 작성한 내용을 실시간으로 모아보려면 Supabase 또는 Firebase와 같은 클라우드 DB 연결이 필요합니다.
- **Supabase**를 활용하면 무료 티어 범위 내에서 간단히 REST API나 SQL 쿼리를 통해 전 세계 방문자의 방명록을 데이터베이스화하여 실시간 공유할 수 있습니다.
